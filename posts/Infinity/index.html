从 AlphaStar 到 LLM Agents

1. 整体训练框架

AlphaStar 的训练流程：

对应现代 LLM：

AlphaStar 现代 LLM / Agent
Human replay Pre-training / Demonstration Data
Imitation Learning Pre-training / SFT
Structured action decoder Tool Calling
Conditioning vector Prompt / Instruction
Reinforcement Learning Post-training
KL penalty Reference Policy Regularization
League / Exploiter Multi-Agent Training / Red Teaming

现代 LLM Agent 面临的许多问题，如工具调用、RL 后策略坍缩、模型盲点和多智能体稳健性，在 AlphaStar 中已经出现。

2. 从传统 Agent 到 LLM Agent

传统 Agent 在环境中执行：

其目标是最大化累计回报：

$$ \max_\theta \mathbb{E}_{\pi_\theta} \left[ \sum_{t=0}^{T}\gamma^t r_t \right]. $$

传统游戏中的 observation、action 和 reward 边界清楚。现代 LLM Agent 则可能连接用户、搜索引擎、Terminal、Browser、文件系统和外部 API。

因此，完整的 LLM Agent 不只是一个 LLM，还包括负责组织上下文、调用工具和控制循环的 Agent Harness。

3. StarCraft 为什么适合研究复杂 Agent？

StarCraft 的主要难点包括：

Partial Observability

Agent 看不到完整地图，需要根据历史信息维护内部状态：

$$ h_t=f_\theta(h_{t-1},o_t). $$

Long Horizon

早期资源分配和建造顺序，可能很久以后才影响胜负，导致 credit assignment 困难。

Combinatorial Action Space

一个 action 通常包含：

  • action type；
  • selected units；
  • target position；
  • delay；
  • repeat。

Strategy 与 Micro-control 并存

Agent 既要规划资源、兵种和进攻时机，也要完成单位移动、技能释放等细粒度操作。

4. Action Modeling 与 Tool Use

StarCraft 的 action 类似一个结构化 API 调用：

AlphaStar 使用 autoregressive decoder 依次预测各个参数：

$$ p(a_t\mid s_t) = \prod_{k=1}^{K} p(a_t^{(k)}\mid a_t^{(<k)},s_t). $$

这与现代 LLM 的 tool calling 很接近：模型需要选择工具、填写参数，并将结果交给环境执行。

区别是：

  • AlphaStar 的 action schema 写入网络结构；
  • LLM 使用自然语言或代码描述不同 API；
  • 同一个 Transformer 可以调用多种工具。

Tool use 本质上也是 action modeling。

5. 为什么不能只用 Pure RL？

StarCraft 的动作空间大、reward 延迟长，随机 policy 很难探索出有效策略。

Pure RL 还可能找到能赢但不理想的漏洞，因此：

$$ \text{Reward Maximization} eq \text{Desired Behavior}. $$

AlphaStar 首先使用人类 replay 做 imitation learning：

$$ \mathcal{L}_{\text{IL}} = - \mathbb{E}_{(s,a)\sim D_{\text{human}}} \left[ \log \pi_\theta(a\mid s) \right]. $$

它帮助 Agent：

  • 学会基本操作和资源管理；
  • 学会常见 build order；
  • 获得多样化策略；
  • 减少 RL 阶段的探索难度。

课程中，单独的 supervised policy 已达到约 900 Elo。

更合理的路线是先学习 human prior，再通过环境交互继续提升，而不是 imitation 与 RL 二选一。

6. Conditioning 与 Prompt

AlphaStar 会从完整 replay 中提取：

  • 最终使用的单位；
  • build order；
  • 玩家水平；
  • 策略风格。

模型被 condition 在这些信息上：

$$ \pi_\theta(a_t\mid o_{\le t},c). $$

这相当于告诉 Agent：

  • 使用某类单位；
  • 采用某种战术；
  • 模仿某种水平或风格。

它与现代 LLM 的 prompt / instruction following 类似。

区别在于 AlphaStar 使用有限的结构化向量，而 LLM 使用开放的自然语言。

7. RL Post-training 与策略坍缩

从 imitation policy 开始做 RL，可以提高胜率，但可能出现 behavior collapse。

Imitation Learning 学到多样的人类策略，而 RL 可能集中到一种暂时有效的策略：

$$ \pi_{\text{diverse}} \rightarrow \pi_{\text{specialized}}. $$

结果是：

  • 原有策略多样性下降；
  • Agent 过度适应当前对手；
  • self-play 中很强；
  • 面对新 counter-strategy 时很脆弱。

这与现代 LLM post-training 中的 mode collapse、capability regression 和 reward over-optimization 类似。

8. Supervised KL Penalty

为了限制 RL policy 偏离 imitation policy，AlphaStar 使用 KL regularization：

$$ \max_\theta \; \mathbb{E}_{\pi_\theta}[R] - \beta D_{\mathrm{KL}} \left( \pi_\theta \Vert \pi_{\text{SL}} \right). $$

作用是：

  • 保留人类策略分布；
  • 减少策略坍缩；
  • 避免过早集中到局部策略；
  • 在提升 reward 的同时维持多样性。

现代 LLM post-training 中的 reference policy constraint 使用了相同思想。

9. Privileged Value Function

StarCraft 是部分可观测的，但训练 critic 时可以让它看到完整地图：

$$ a_t\sim\pi_\theta(a_t\mid o_{\le t}), $$

$$ V_\phi(s_t^{\text{full}}) \approx \mathbb{E}[R_t\mid s_t^{\text{full}}]. $$

即：

  • Actor 只能使用真实可见信息；
  • Critic 可以使用完整状态。

这样能提高 value estimation 的准确度、降低训练方差并改善 credit assignment。

对现代 Agent 来说，verifier 或 critic 也可以使用完整 trajectory、最终环境状态或测试结果。

10. AlphaStar RL 与 LLM Post-training 的差异

Reward 更难定义

StarCraft 的胜负相对明确，而 LLM 的开放任务常常没有唯一标准，例如“写一首好诗”或“给出有帮助的建议”。

Post-training 规模

讲者认为，现代 LLM 的 compute 仍然过度集中于 pre-training。AlphaStar 则在 RL 与 League interaction 中进行了大量训练，post-training 达到约：

$$ 10^{10} $$

个 training steps。

这说明未来 Agent 的提升可能来自更大规模的环境交互，而不只是更大的 pre-trained model。

11. 为什么需要 Multi-Agent Learning？

一个 Agent 可能总体 Elo 很高，却仍存在简单而严重的弱点。

LLM 也类似：

  • 能解决复杂数学问题，却在简单字符计数上失败；
  • benchmark 分数高，却容易被特殊 prompt 攻击；
  • 平均性能强，但 worst-case robustness 较差。

Multi-Agent Learning 用于主动发现：

  • blind spot；
  • adversarial behavior；
  • counter-strategy；
  • distribution shift；
  • 可迁移 failure mode。

12. Non-transitivity

策略强弱不一定满足简单排序，可能出现：

$$ \pi_A \succ \pi_B, \qquad \pi_B \succ \pi_C, \qquad \pi_C \succ \pi_A. $$

最典型的是 Rock-Paper-Scissors。

这说明单一 Elo 或 leaderboard score 无法完整表示 Agent 能力。两个平均分接近的模型，可能擅长不同任务，也可能具有完全不同的弱点。

因此，Agent evaluation 还需要关注：

  • worst-case performance；
  • 能力覆盖；
  • 针对性攻击；
  • 面对新对手时的稳定性。

13. 为什么 Naive Self-play 会失败？

AlphaStar 中存在类似 Rock-Paper-Scissors 的单位克制：

在 naive self-play 中，Agent 逐渐只使用 Void Ray。由于双方都采用同一策略，训练对局看起来很强，但 Stalker 已经从训练分布中消失。

外部对手只需使用 Stalker，就能轻易击败 Agent。

Self-play 中变强，不代表面对真实、多样化对手时也足够 robust。

14. AlphaStar League

AlphaStar 使用由大量不同 policy 组成的 population，包括：

  • Main Agents；
  • 历史 checkpoints；
  • Main Exploiters；
  • League Exploiters；
  • 不同策略和风格的 Agent。

Main Agent

最终希望部署的 Agent。它与当前 Agent、历史 checkpoint 和各类 exploiter 对战，不断修复弱点。

Main Exploiter

专门针对某个 Main Agent：

  1. 从 imitation prior 初始化；
  2. 固定 Main Agent；
  3. 学习击败它；
  4. 胜率足够高后保存 checkpoint；
  5. 将该 checkpoint 加入 League；
  6. Main Agent 再学习防御。

这类似针对单个 LLM 的 adaptive red teaming。

League Exploiter

目标不是击败单一 Main Agent，而是找到对整个 population 都有效的策略。

类型 目标
Main Exploiter 找到某个 Agent 的特定弱点
League Exploiter 找到多个 Agent 的共同弱点

15. Matchmaking

League 中有大量 Agent，需要决定下一局选择谁作为 opponent。

如果选择一个几乎必胜于自己的对手，Main Agent 会一直输，学习信号反而有限。

更有效的方法是选择胜率接近 $50\%$ 的对手。

二元胜负结果的 entropy 为：

$$ H(p) = -p\log p-(1-p)\log(1-p). $$

当：

$$ p=0.5 $$

时 entropy 最大，因此 50/50 opponent 通常能提供更丰富、稳定的学习信号。

16. Conditioning Exploiters

如果所有 Exploiter 都自由寻找获胜方式，它们可能反复发现同一种弱点。

因此，可以给 Exploiter 添加限制：

这样可以主动扩大 counter-strategy 的多样性。

对 LLM 来说，可以用 prompt 控制不同：

  • 攻击方式；
  • 语言；
  • 工具；
  • 任务类型；
  • 推理方式。

Prompt 不只用于控制输出，也可以用于扩大训练中的 interaction distribution。

17. 对现代 LLM Agent 的启发

Tool Use

Tool call 应被视为 policy action，不只评估最终答案，也要评估完整 tool trajectory。

Human Prior + Experience

比 pure imitation 或 pure RL 更合理。

KL Regularization

Post-training 需要在提升 reward 和保留原能力之间取得平衡。

Privileged Critic

Critic 或 verifier 可以使用完整 trajectory、测试结果和最终环境状态。

Population Training

可以维护：

  • 不同 checkpoint；
  • 不同 system prompt；
  • adversarial users；
  • task generators；
  • critics；
  • target-specific exploiters；
  • general red-team agents。

18. 与 LLM Multi-Agent 的关键差异

用户通常是协作方

StarCraft 是 zero-sum game，但 LLM Agent 与普通用户通常是合作关系。

同时存在对抗用户

开放系统仍会遇到 jailbreak、prompt injection、恶意工具调用和安全边界绕过。

Reward 更模糊

LLM 任务往往同时考虑正确性、有用性、安全性、成本和用户满意度，这些目标还可能冲突。

Prompt Space 更大

AlphaStar 的 conditioning 有限，而自然语言 prompt 几乎可以表达任意任务和约束，使 LLM population 更复杂。

19. 对 Agent Evaluation 的启发

Agent evaluation 不应只看平均分，还应评估：

  • worst-case performance；
  • distribution shift；
  • adaptive attack；
  • transferable failures；
  • 是否遗忘历史 failure mode；
  • 面对新 opponent 的适应能力。

可以构造：

  • Target-specific Exploiter： 针对一个固定模型持续寻找失败样例；
  • League-level Exploiter： 针对多个模型寻找共同 failure mode。

这比固定 benchmark 更接近持续、适应性的 Agent evaluation。

20. 总结

  1. Tool use 本质上是 action modeling。

  2. Human prior 与 RL 不是二选一。
    人类数据提供合理起点，环境交互提供新经验。

  3. RL post-training 可能破坏原有策略多样性。
    KL regularization 可以限制策略坍缩。

  4. Naive self-play 不等于 robustness。
    Agent 可能只在自己创造的狭窄策略分布中变强。

  5. Population-based training 可以主动发现弱点。
    Main Exploiter、League Exploiter、历史 checkpoint 和 matchmaking 共同构成持续进化的环境。

  6. 未来 Scaling 不一定只来自更大的模型。

$$ \text{Scaling} = \text{Model Scale} + \text{Interaction Scale} + \text{Population Diversity}. $$

AlphaStar 的经验表明，更强的 Agent 不只是更大的 neural network，还需要合理的人类先验、长期环境交互、稳定的 post-training 和多样化的对手分布。
