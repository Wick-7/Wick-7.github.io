语言模型内部运作

第一部分：从 Prompt 到下一个 Token

2. Tokenization 与 Embedding

输入文字先被 Tokenizer 切成 Token，每个 Token 对应一个整数 ID。课件为了方便，假设一个汉字就是一个 Token；真实情况中，一个 Token 可能包含多个汉字，一个汉字也可能被拆开。

Embedding Table 将 Token ID 转成向量：

$$ E\in\mathbb{R}^{V\times d},\qquad h_i^{(0)}=E[x_i] $$

其中 $V$ 是词表大小，$d$ 是向量维度。Embedding Table 本身也是模型参数。

初始 Token Embedding 的特点：

  • 相同 Token 的向量完全相同；
  • 语义相关的 Token 通常更接近；
  • 多义词还没有结合上下文，因此会混合不同含义。

例如，$apple$ 可能同时接近水果相关词和 Apple 公司相关词。

3. Layer by Layer

Token Embedding 会依次经过多层 Transformer：

$$ H^{(\ell)}=\operatorname{Layer}_{\ell}\left(H^{(\ell-1)}\right) $$

每层输入一串向量，也输出同样长度的一串向量。经过层处理后，向量融合了上下文，因此也称为：

  • Contextualized Embedding；
  • Hidden / Latent Representation；
  • Hidden State。
类型 是否结合上下文 相同 Token 的表示
Token Embedding 否 相同
Contextualized Embedding 是 可随语境变化

例如，水果含义和公司含义的 $apple$ 初始向量相同，但经过多层后会逐渐分开。

4. LM Head、Logits 与 Softmax

模型取最后一层、最后一个位置的 Representation，通过 LM Head 映射到整个词表：

$$ z=W_{\mathrm{LM}}h_n^{(L)} $$

$z$ 中每一维对应一个 Token 的分数，称为 Logit。Logit 可以是任意实数，并不是概率。

许多模型会让 LM Head 与输入 Embedding Table 共用参数。此时可以把 Unembedding 理解为：最终 Representation 与每个 Token Embedding 计算匹配分数。

$$ z_i=E_i^\top h_n^{(L)} $$

Softmax 将 Logits 转为便于采样的分布：

$$ p_i(T)=\frac{\exp(z_i/T)}{\sum_j\exp(z_j/T)} $$

  • $T$ 大：分布更平均，输出更随机；
  • $T$ 小：分布更集中，输出更保守。

实际解码还可能结合 Top-k、Top-p 等方法。这里的概率主要用于排序和采样，不必理解为完全校准的真实概率。

第二部分：每层 Representation 表示什么

5. Representation Space

Embedding / Representation 位于高维空间中：

  • 含义相近的内容可能聚集；
  • 某些方向可能对应特定语义变化；
  • 不同模型、不同层中的方向含义未必相同。

$$ \operatorname{Emb}(\text{冷})- \operatorname{Emb}(\text{cold})+ \operatorname{Emb}(\text{small}) \approx \operatorname{Emb}(\text{小}) $$

这种向量加减有时成立，但不是所有模型都能稳定复现。

6. 投影到低维空间

Representation 通常有几千维，可以寻找特定低维子空间进行观察。

Structural Probe

研究发现，BERT 某些中间层的 Representation 经特定投影后，可以呈现接近句法树的结构。重点是：

  • 不是任意投影都能得到句法树；
  • 中间层通常包含较明显的句法信息；
  • 浅层和最后层未必最适合表示句法结构。

7. Activation Steering

除了观察 Representation，还可以直接修改它，看模型行为如何变化。相关名称包括 Representation Engineering、Activation Engineering 和 Activation Steering。

以拒绝行为为例，计算“会拒绝”和“不会拒绝”两类 Prompt 在某层的平均 Representation，二者之差作为候选拒绝方向：

$$ r^{(\ell)}= \mu_{\mathrm{reject}}^{(\ell)}- \mu_{\mathrm{answer}}^{(\ell)} $$

推理时加入或减去该方向：

$$ \widetilde h^{(\ell)}=h^{(\ell)}+\alpha r^{(\ell)} $$

  • 加入可能增强拒绝倾向；
  • 减去可能削弱拒绝倾向；
  • 需要逐层实验，寻找最有效的层。

类似方法也可寻找 Sycophancy，即过度迎合，相关的方向。

局限：均值差可能混入其他因素，一个行为也未必只由单一方向控制；干预成功不等于完全解释了模型机制。

8. Logit Lens

正常情况下只有最后一层经过 LM Head。Logit Lens 把每一层的 Hidden State 都送入 LM Head：

$$ z^{(\ell)}=W_{\mathrm{LM}}h_n^{(\ell)} $$

这样可以观察模型在不同层最倾向输出哪个 Token。

翻译案例中，输入法语 $fleur$ 并要求输出中文。模型中间层先出现英文 $flower$，后层才变成中文“花”。这提示某些多语言模型可能先形成偏英文的中间表示。

但 Logit Lens 只是观察工具：中间层本来未必适合直接被最终 LM Head 解码，Top-1 Token 也不能代表完整的内部信息。

9. Patchscopes

Logit Lens 只能将 Representation 解释成一个 Token；Patchscopes 则尝试生成完整句子：

  1. 从 Source Prompt 的某层取出 Representation；
  2. 构造 Target Prompt，例如“请简单介绍 X”；
  3. 用 Source Representation 替换 $X$ 的 Representation；
  4. 从该层继续生成文字。

改变 Target Prompt，可以从不同角度解释同一个 Representation。

11. Self-Attention

更新某个 Token 的 Representation 时，Self-Attention 要完成两件事：

  1. 找到上下文中哪些 Token 与它相关；
  2. 将这些 Token 的信息加进来。

Query、Key、Value：

$$ Q=HW_Q,\qquad K=HW_K,\qquad V=HW_V $$

  • Query：当前 Token 想找什么；
  • Key：每个 Token 可以如何被匹配；
  • Value：被关注后实际读取的内容。

核心计算：

$$ A=\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right), \qquad O=AV $$

即先用 Query 和 Key 算相关程度，再用得到的权重对 Value 加权求和。最后通过 Residual Connection 保留原信息：

$$ H'=H+O $$

12. 位置信息与 Multi-Head Attention

只有 Token Embedding 时，模型不知道 Token 的位置，因此需要位置编码。

一种简单方法是加入 Positional Embedding：

$$ \widetilde h_i=h_i+p_i $$

LLaMA 等模型通常使用 RoPE，根据位置旋转 Query 和 Key，使 Attention 分数包含相对位置信息。

单个 Attention Head 通常只擅长某类关系，例如数量、形容词或指代，因此模型使用多个 Head：

$$ \operatorname{MHA}(H)= \operatorname{Concat}(\operatorname{head}_1,\ldots,\operatorname{head}_m)W_O $$

课堂实作中，LLaMA 使用 Grouped-Query Attention：多个 Query Head 共用较少的 Key / Value Head，从而减少参数量和 KV Cache。

13. Causal Attention 与复杂度

自回归语言模型在第 $i$ 个位置只能关注自己和左侧 Token，不能读取未来内容：

$$ M_{ij}= \begin{cases} 0,&j\le i\\ -\infty,&j>i \end{cases} $$

因此 Attention Heatmap 的右上三角为 0。

长度为 $n$ 的序列需要计算 $n × n$ 的 Attention Matrix，主要开销约为：

$$ \mathcal O(n^2) $$

这也是长上下文计算成本较高的原因之一。

14. Feed-Forward Network

Self-Attention 负责 Token 之间的信息交换；FFN 对每个位置独立应用相同的非线性变换：

$$ \operatorname{FFN}(x)=W_2\sigma(W_1x+b_1)+b_2 $$

通常先扩维再压回原维度：

$$ d\rightarrow d_{\mathrm{ff}}\rightarrow d $$

课件用 ReLU 说明激活函数：

$$ \operatorname{ReLU}(x)=\max(0,x) $$

现代 LLM 常使用 SiLU、GeLU 或 SwiGLU。FFN 也可以被解释为某种 Key-Value Memory：第一层检测模式，第二层把对应信息写回 Residual Stream。

矩阵变换中的单个输出维度可以看作一个 Neuron，即“加权求和 + Bias + Activation”。

15. 分析 Embedding 与 Hidden State

可计算 Token Embedding 之间的 Cosine Similarity：

$$ \operatorname{cos}(u,v)= \frac{u^\top v}{\lVert u\rVert_2\lVert v\rVert_2} $$

课堂中，$apple$ 会接近不同大小写形式、中文“苹果”、$iPhone$、$MacBook$ 或 $Cupertino$ 等词，说明 Embedding 编码了统计语义关系，但个别近邻不一定容易解释。

课堂实验表明：

  • 相同 Token 在第 0 层完全相同；
  • 经过 Transformer 后，会因左侧上下文不同而分开；
  • 同义上下文中的 $apple$ 相似度较高，不同含义则随层数增加逐渐分离；
  • 跨层比较时要考虑不同层本身的分布差异，不能只看原始 Cosine Similarity。

16. 实作 Logit Lens 与 Attention

Logit Lens：

输入“天气”时，中间层可能出现 $weather$、$forecast$ 等英文词，最后层才形成目标中文 Token。这与前面的多语言中间表示现象一致，但不能推广为所有模型都使用英文思考。

Attention Weight：

在句子 $The apple is green. What color is the apple?$ 中，部分 Head 会让后面的 $apple$ 或 $color$ 关注前面的 $green$ 和 $apple$。右上三角为 0，来自 Causal Mask。

Attention Weight 只是某个 Head 在本次计算中的信息路由，不能直接等同于完整解释。很多 Head 还会关注 BOS Token，这可能是 Attention Sink：当没有明显相关信息时，BOS 成为默认落点。
