Multi-Agent AI：Self-Play、博弈均衡与多智能体协作

0. 课程核心问题

Self-play 曾推动 AlphaGo、Poker AI、Dota、StarCraft 等系统取得突破，但类似的递归式自我改进（recursive self-improvement） 尚未在 LLM 中以同样清晰、稳定的方式出现。

讲者用 AlphaGo 与 LLM 的发展路径进行类比：

  1. Pretrain on high-quality human data

    • AlphaGo：学习高质量人类棋局；
    • LLM：在大规模互联网文本上进行 next-token prediction。

  2. Enable large-scale inference compute

    • AlphaGo：Monte Carlo Tree Search（MCTS）；
    • LLM：Chain-of-Thought、test-time compute。

  3. Recursive self-improvement

    • AlphaGo：通过大量 self-play 不断变强；
    • LLM：这一环节仍缺少通用而可靠的方法。

讲者的核心判断是：

人们对 self-play 的直觉往往过度来自 Chess、Go 等 two-player zero-sum perfect-information games。
一旦进入 imperfect-information、general-sum 或需要与人类合作的环境，许多理想性质都会消失。

本节结论： Self-play 并不是一种脱离环境结构后仍然普遍有效的“通用自我改进算法”。它的成功高度依赖博弈类型、优化目标以及对手分布。

1. “最强玩家”到底是什么意思？

讲者首先提出一个问题：谁是更强的 Poker player？

  • Option 1：与任何其他玩家进行足够多次 head-to-head 对局，都能取得优势；
  • Option 2：在真实 Poker population 中，一年内赚到最多的钱。

这两个目标分别对应：

目标 含义 关注点
Minimax Equilibrium 面对任意 best response 都不被击败 最坏情况鲁棒性
Population Best Response 针对现实玩家群体获得最大收益 对特定 population 的适应和利用

在 Chess、Go 等场景中，两者通常高度一致：最难被击败的玩家往往也是现实比赛表现最好的玩家。

但在 Poker 中，两者可能不同：

  • 一位玩家可能非常接近均衡，面对顶尖对手也很稳健；
  • 另一位玩家可能更擅长发现普通玩家的弱点，从而赚取更多收益；
  • 后者面对几乎没有明显弱点的顶尖玩家时，反而可能稍弱。

因此，“解决一个博弈”默认被定义为求得 minimax equilibrium，本身就是一个很强的目标假设，不一定等于真实部署时想要优化的指标。

2. Minimax Equilibrium 与 Exploitability

2.1 Minimax Equilibrium

在 two-player zero-sum game 中，双方收益满足：

$$ u_1(\pi_1,\pi_2)=-u_2(\pi_1,\pi_2). $$

Minimax value 可写为：

$$ v^* = \max_{\pi_1}\min_{\pi_2}u_1(\pi_1,\pi_2) = \min_{\pi_2}\max_{\pi_1}u_1(\pi_1,\pi_2). $$

Minimax equilibrium 是一组策略，使任何玩家都无法通过单方面改变策略提高自己的期望收益。

在 two-player zero-sum game 中，使用 minimax equilibrium 可以保证：

即使对手知道我们的完整策略，也无法让我们在期望意义上输掉比赛。

2.2 Exploitability

Exploitability 衡量当前策略与 minimax equilibrium 的距离，即面对 best response 时会损失多少价值。

对玩家 1 的固定策略 $\pi_1$，可直观写为：

$$ \operatorname{Exploitability}(\pi_1) = v^* - \min_{\pi_2}u_1(\pi_1,\pi_2). $$

  • Exploitability 越大，策略中可被针对的弱点越明显；
  • 达到 equilibrium 时，exploitability 为 $0$。

2.3 Rock-Paper-Scissors 示例

若始终出 Rock：

  • 对手的 best response 是始终出 Paper；
  • 我方每轮都输；
  • Exploitability 为 $1$。

即使策略看起来更复杂，例如：

  • 第 1、2 轮出 Rock；
  • 第 3 轮根据历史改出 Scissors；

只要策略规则是公开的，对手仍可构造完整的 best response，因此 exploitability 依旧可能为 $1$。

2.4 均衡分析中的关键假设

均衡分析假设：

  • 策略本身是 common knowledge；
  • 随机过程的具体结果不是 common knowledge，例如对手不知道 random seed。

这一假设看起来很强，但对公开部署的 AI 有现实意义：当系统被大量使用时，潜在弱点最终可能被找到，因此应当假设攻击者充分了解策略结构。

Rock-Paper-Scissors 的均衡策略是：

$$ \pi^* = \left( \frac{1}{3}, \frac{1}{3}, \frac{1}{3} \right). $$

对手知道我们以等概率随机选择 Rock、Paper、Scissors，但不知道某一轮具体采样结果，因此最多只能与我们打平。

2.5 为什么 Poker 均衡仍能赚钱？

在 Rock-Paper-Scissors 中，对手面对均衡策略时采取很多策略都能打平，因此我们通常不会主动获得正收益。

Poker 更复杂：

  • 有些动作本身具有 negative expected value；
  • 对手偏离均衡时会犯错并损失价值；
  • 因为是 zero-sum game，对手的损失会转化为我方收益。

因此现实中的基本思路是：

  1. 先使用接近 minimax equilibrium 的策略确保自身稳健；
  2. 对手犯错时自然获利；
  3. 长期观察到对手的稳定弱点后，再谨慎偏离均衡进行 exploit。

本节结论： Minimax 主要解决“最坏情况下不输”的问题；Population Best Response 解决“面对真实对手分布如何赢得更多”的问题。

3. Two-Player Zero-Sum Games 中的 Self-Play

Self-play 指 agent 通过不断与自己的副本对局而逐渐改进。

在有限、平衡的 two-player zero-sum game 中，如果使用 sound self-play algorithm，则平均策略可以在理论上收敛到 minimax equilibrium。

这带来一个非常强的性质：

  • 初始策略甚至可以完全随机；
  • 不需要人类专家数据；
  • 只要具有足够的 memory 与 compute，就可以通过 self-play 逼近不可被击败的策略。

不过这里必须强调两个限定：

  1. 不是任意 naive self-play 都能可靠收敛；
  2. 理论上的均衡收敛不等于有限神经网络和有限算力下的实际安全。

4. Perfect-Information Games：相对简单，但仍可能被利用

在 two-player zero-sum perfect-information game 中，self-play 很接近两个独立的 single-agent RL 过程。

例如使用 PPO：

  • 每一方都将另一方视作环境的一部分；
  • 在具有持续探索、无限容量和充分训练的理想条件下，可以收敛到 minimax。

但实际系统仍可能被 adversarial strategy 击败，原因包括：

  • 神经网络只能近似 minimax policy；
  • 训练探索有限，未覆盖特殊状态；
  • 模型容量和计算资源有限；
  • 寻找一个 exploit 通常比防御所有 exploit 更容易。

课件以强 Go bot 的 adversarial policy 为例说明：系统在正常 benchmark 或常规对局中非常强，并不意味着它已经真正达到低 exploitability。

本节结论： 常规胜率高、平均性能强与 worst-case robust 并不是同一个目标。

5. Imperfect-Information Games 为什么更难？

5.1 PPO 不保证找到正确的混合策略

课件构造了一个不对称的 Rock-Paper-Scissors+，
其 equilibrium mixed strategy 为：

$$ \pi^* = (0.4,0.4,0.2), $$

即：

  • Rock：$40\%$；
  • Paper：$40\%$；
  • Scissors：$20\%$。

PPO 并没有机制保证策略概率收敛到这一特定比例，因此在实验中会持续振荡，并被 optimal opponent exploit。

5.2 不完全信息的核心困难

imperfect-information game 的关键性质是：

一个动作的 value 不仅取决于动作本身，还取决于它被选择的概率。

这与 standalone Chess/Go 不同：

  • Chess 中某个开局走法是否好，通常不强依赖其在策略中的频率；
  • Poker 中 bluff 的价值则高度依赖 bluff frequency；
  • bluff 太少无法获得足够收益；
  • bluff 太多会被对手识别并频繁 call。

所以算法必须学习的不只是：

“选择哪个动作？”

还必须学习：

“以多大概率选择每个动作？”

6. Fictitious Play

Fictitious Play 的基本过程：

  1. 任意初始化每个玩家的策略；
  2. 第 $t+1$ 轮，每个玩家计算对手历史平均策略的 best response；
  3. 使用所有历史策略的平均值作为最终策略。

设对手在前 $t$ 轮的平均策略为：

$$ \bar{\pi}_{-i}^{(t)} = \frac{1}{t} \sum_{\tau=1}^{t} \pi_{-i}^{(\tau)}. $$

玩家下一轮采用：

$$ \pi_i^{(t+1)} \in \operatorname{BR} \left( \bar{\pi}_{-i}^{(t)} \right). $$

在 two-player zero-sum game 中，历史平均策略会收敛到 minimax equilibrium。

Fictitious Play 的问题是：每轮都使用精确 best response，策略变化很激烈，收敛速度通常较慢。

7. Regret Matching、Hedge 与 CFR

7.1 Regret Matching

与 Fictitious Play 总是选择 exact best response 不同，Regret Matching 根据各动作的正 regret 分配概率，相当于一种 regularized best response。

动作 $a$ 的累计 regret 可理解为：

$$ R_t(a) = \sum_{\tau=1}^{t} \left[ u(a,\pi_{-\!i}^{(\tau)}) - u(\pi_i^{(\tau)},\pi_{-\!i}^{(\tau)}) \right]. $$

下一轮策略通常取：

$$ \pi_{t+1}(a) = \frac{[R_t(a)]_+} {\sum_b [R_t(b)]_+}, $$

其中：

$$ [x]_+=\max(x,0). $$

含义是：

  • regret 小于等于 $0$ 的动作不分配概率；
  • 正 regret 越大的动作，下一轮选择概率越高；
  • 不会像 exact best response 一样突然把全部概率压到单一动作上。

7.2 Hedge

Hedge 也通过 regularization 平滑更新动作概率。它与 online learning、gradient-based optimization 存在很强的联系。

7.3 CFR 与 Linear CFR

在 extensive-form imperfect-information game 中，可以在每个 information set 上进行 regret minimization，这构成 Counterfactual Regret Minimization（CFR）的核心思想。

  • 普通 CFR：策略概率围绕均衡持续振荡，平均策略缓慢靠近 $1/3$；
  • Linear CFR：对后期迭代给予更大权重，明显更快稳定到均衡附近。

7.4 Libratus

课件以 Libratus 为代表案例：

  • 2017 年与四位顶尖 heads-up no-limit Texas Hold’em 职业玩家对战；
  • 共进行 120,000 hands、持续 20 天；
  • 每位人类玩家都单独输给 Libratus；
  • 训练来自 self-play；
  • 未使用人类 Poker 数据；
  • 主要突破来自 imperfect-information search 与 equilibrium computation，而非 deep neural network。

这说明：在 two-player zero-sum imperfect-information game 中，只要目标仍是 minimax equilibrium，self-play 与 regret minimization 依然非常强大。

7.5 传统方法的限制

Fictitious Play、Regret Matching、Hedge 的常见问题：

  • 在 single-agent RL 上表现较差；
  • 第一轮通常从随机策略开始；
  • policy improvement 较慢；
  • 可能需要大量迭代；
  • 适合 equilibrium finding 的算法，不一定适合普通 reward maximization。

7.6 New Last-Iterate Algorithms

课件提到一些新的算法，尝试同时获得：

  • single-agent RL 中的良好优化性能；
  • multi-agent game 中向 minimax 的快速收敛。

代表包括：

  • Regularized Nash Dynamics；
  • Magnetic Mirror Descent。

其共同思路是：在类似 Hedge 的更新上，进一步加入对某个 “magnet policy” 的 regularization，改善 last-iterate behavior。

8. 为什么前半部分几乎没有提到 LLM？

在 two-player zero-sum minimax equilibrium 中，cheap talk communication 没有价值。

证明直觉如下：

  • 所有 minimax equilibria 都对应同一个唯一 value：

$$ (v_1,-v_1). $$

  • 假设 Player 1 发送一条不具有约束力的信息：
    • 如果信息能增加 $v_1$，它必然损害 Player 2，Player 2 会忽略；
    • 如果信息会降低 $v_1$，Player 1 就不应发送；
    • 剩下的情况只有信息不改变 equilibrium value。

因此在纯 two-player zero-sum minimax setting 中，语言交流不是关键变量。

LLM 的语言能力真正开始重要，是在以下场景中：

  • 多于两个玩家；
  • general-sum game；
  • 合作、协商、联盟、信任；
  • 玩家需要根据对方的偏好与行为分布进行适应。

9. 离开 Two-Player Zero-Sum：为什么需要 Human Data？

讲者提出了课程中最重要、也较有争议的观点：

若目标是学习如何与人类合作，那么完全避免使用 human data 是一条 dead end。

原因并不是 self-play 完全无法产生合作，而是：

  • general-sum game 往往有多个 equilibrium；
  • self-play 只能收敛到其中某一种约定；
  • 该约定未必与人类社会实际采用的约定兼容；
  • 如果最终评价对象是人类 population，就必须了解该 population。

10. Ultimatum Game

本节结论： 果部署目标是与人类合作，“数学上的 equilibrium” 不等于 “人类会接受的 equilibrium”。

11. Diplomacy：Self-Play 在 General-Sum Game 中的局限

11.1 游戏特点

Diplomacy 是一个七人策略与谈判游戏：

11.2 Support 为什么关键？

若两个单位以 $1$ 对 $1$ 争夺同一区域，双方都会失败并退回。

若一个单位获得另一单位支持，则形成 $2$ 对 $1$，可以成功进入目标区域。

更重要的是，support 可以来自其他玩家，因此需要：

  • 请求帮助；
  • 交换利益；
  • 建立承诺；
  • 判断对方是否会真正执行；
  • 防止对方背叛。

11.3 DORA：从零开始 Self-Play

DORA 在 no-press Diplomacy 中不使用自然语言，只通过动作进行隐式协调，并采用类似 AlphaZero 的 pure self-play。

在 two-player no-press Diplomacy 中：

$$ \text{Win Rate} = 86.5\%\pm6.1\% $$

对人类专家表现出明显优势。

但在 seven-player setting 中出现了 equilibrium mismatch：

  • 1 个 SearchBot + 6 个 DORA：
    • SearchBot 的获胜比例约为 $1.1\%\pm0.4\%$；
  • 1 个 DORA + 6 个 SearchBot：
    • DORA 的获胜比例约为 $11.0\%\pm1.5\%$。

七人完全对称时，平均基准约为：

$$ \frac{1}{7}\approx14.3\%. $$

这组结果意味着：

  • SearchBot 在 DORA population 中表现很差；
  • DORA 在 SearchBot population 中同样低于平均；
  • 两类 agent 各自学到了一套内部一致的 coordination convention；
  • 一旦进入另一种 population，它们便无法正确理解或利用对方。

11.4 多个 Equilibria 的问题

在 two-player zero-sum game 中，不同训练方法只要都逼近 minimax，彼此对战通常仍能保持稳健。

在 general-sum 或多玩家博弈中：

  • 可能存在大量互不兼容的 equilibria；
  • self-play 的结果取决于初始化、训练群体和历史路径；
  • “与自己的副本合作得很好”不代表“与陌生 agent 或人类合作得好”。

因此 minimax equilibrium 在一般博弈中不再是足够有意义的部署目标。

更现实的目标是：

Population Best Response

但随后必须回答：

Population 到底是谁？

  • 其他自己的副本？
  • 某种固定 AI 群体？
  • 现实中的人类？
  • 某个特定文化或组织中的用户？

只要目标 population 是人类，就需要关于人类的行为数据。

12. 将 Humans 视为 Environment 的一部分

讲者推荐的方案分为三步。

Step 1：Human Imitation Model

收集大量人类行为数据，训练 imitation model：

$$ \hat{\pi}_{H}(a\mid s) \approx \pi_{H}(a\mid s). $$

目标不是先学习“理论最优人类”，而是尽可能准确地预测：

在给定状态下，真实人类会怎么做。

Step 2：Scale Inference-Time Compute to Better Model Humans

增加 inference-time compute，用于更准确地模拟人类：

  • 推测其意图；
  • 预测其行动；
  • 估计其可能接受的协作方案；
  • 建模其谈判与反应。

这里的重点不是单纯提高 imitation model 自己的游戏分数，而是提高对 human behavior distribution 的建模质量。

Step 3：在人类模型环境中进行 RL

将 human imitation models 作为环境中的其他玩家，再训练目标 agent：

$$ \pi_A^* = \arg\max_{\pi_A} \mathbb{E}_{\pi_H\sim\hat{P}_H} \left[ R(\pi_A,\pi_H) \right]. $$

本质上是学习对 human population 的近似 best response。

12.1 Diplodocus

采用这一思路训练的 Diplodocus：

  • 参加 200-game real-human no-press Diplomacy tournament；
  • 每个 bot 进行 50 局；
  • Diplodocus-High 最终排名第 1；
  • pure self-play 的 DORA 仅排在约第 32 名，低于平均人类水平。

这说明：在真实 human population 中，准确建模人类行为比仅在自己的 self-play equilibrium 中变强更重要。

12.2 CICERO

CICERO 被匿名投入 online natural-language Diplomacy league：

  • 位列玩家 top $10\%$；
  • 在至少完成 5 局的 19 位玩家中排名第 2；
  • 平均得分超过普通人类平均分的 2 倍。

CICERO 的关键不是单纯让 LLM 进行自由聊天，而是将：

  • strategic planning；
  • human behavior modeling；
  • natural language negotiation；

结合在同一个决策系统中。

12.3 Hanabi

在 Hanabi 协作实验中，human modeling + RL 的方法也展现出更好的跨人群合作表现。

课件表格中，对所有 56 位 testers：

搭档策略 平均得分
Human Experts $14.54\pm1.47$
BR-BC $16.73\pm1.27$
piKL3 $\mathbf{17.18\pm1.28}$

说明针对 human behavior distribution 进行训练，有助于 agent 与不同水平的人类玩家协调。

13. 关于 Self-Play 的阶段性结论

  1. Two-player zero-sum games 是特殊情形。
  2. 在该情形中：
    • minimax 有明确含义；
    • equilibrium value 唯一；
    • sound self-play 具有收敛保证；
    • 不一定需要 human data。
  3. 在一般博弈中：
    • AlphaGo-style self-play 不会自动收敛到唯一的“最优策略”；
    • 多个 equilibria 可能彼此不兼容；
    • 真实效果依赖对手 population。
  4. 若目标是和人类合作：
    • 应显式收集 human data；
    • 建模 human behavior；
    • 在近似人类 population 中优化策略。

Part II：Agent-Agent LLM Cooperation

14. 为什么要研究 Agent-Agent Cooperation？

当目标变成 AI agents 彼此协作时，不一定需要 human data，因为最终 population 本身就是 AI。

讲者给出两个主要动机：

  1. Latency / 并行化推理
  2. Diversity / 专业分工

15. Test-Time Compute 与串行 CoT

Reasoning model 的性能会随着 test-time compute 增加而提升：

  • 模型思考更久，通常能处理更难的问题；
  • 新一代模型也会让整条 compute-performance curve 左移；
  • 即用更低成本达到旧模型更高计算预算下的性能。

课件使用 AIME、Codeforces 等图表说明 reasoning performance 的快速提升。

但 serial Chain-of-Thought 存在根本问题：

每一步依赖前一步，因此难以无限并行。

当推理持续数小时、数天甚至更长时，latency 会成为瓶颈。

16. Parallel Test-Time Scaling

16.1 Consensus

对同一问题独立采样多次，然后选择最常见答案。

例如：

  1. 对数学题采样 64 份回答；
  2. 提取每份回答的最终答案；
  3. 使用多数投票作为输出。

优势：

  • 容易并行；
  • 可以显著降低 wall-clock latency；
  • 对短答案、明确答案的任务效果较好。

局限：

  • 输出必须容易比较；
  • 长文本几乎不会逐字相同；
  • 多数答案不一定正确；
  • 多个样本可能产生相关错误。

16.2 Best-of-N

生成 $N$ 个候选，然后通过 scorer、verifier 或 unit test 选出最优答案：

$$ y^* = \arg\max_{y_i,\ i=1,\ldots,N} s(y_i). $$

特别适合容易验证的任务，例如：

  • Sudoku；
  • 编程题与 unit tests；
  • 可自动检查的数学答案；
  • 具有明确 reward function 的任务。

局限是必须具备可靠 verifier。若答案质量难以自动判断，Best-of-N 可能只是把生成问题转移成评价问题。

16.3 并行方法的 Trade-off

方法 推理结构 优势 局限
Serial CoT 串行 单位 compute 更有效，可逐步深化 latency 高
Consensus 并行采样 + 投票 简单、低延迟 只适合可对齐的短答案
Best-of-N 并行采样 + 评分 有 verifier 时非常有效 依赖评分器，compute efficiency 较低

课件总结：

Parallel methods 往往具有更低 latency，但 compute efficiency 更低。

17. Diversity 与 Routing

一个强 reasoning model 也许可以完成大数乘法，但最合理的行为通常是调用 calculator。

同样地：

  • 最大、最强的模型不一定是每个任务的最佳执行者；
  • 不同模型可能在 coding、math、vision、retrieval、planning 等任务上具有不同优势；
  • 某些模型更快、更便宜；
  • 某些模型更专业或更可靠。

因此可以设计 router：

  1. 接收 query；
  2. 判断任务类型；
  3. 选择最合适的 model / agent；
  4. 将任务转交；
  5. 汇总并返回结果。

讲者认为：

Routing 本身已经是一种 multi-agent AI。

这里的 multi-agent 不一定意味着多个 agent 长时间对话，也可以是：

  • 一个 coordinator；
  • 多个 specialized workers；
  • 按任务动态选择执行者。

18. 当前 Multi-Agent Collaboration Scaffold 的问题

当前很多 multi-agent system 依赖复杂 scaffold：

  • 主 agent 分解任务；
  • 多个 subagents 分别处理子任务；
  • memory 保存中间结果；
  • context compression 压缩历史；
  • citation agent 或 verifier 检查输出；
  • coordinator 合并结果。

这些系统在部分场景中有效，但通常存在：

  • heavily engineered；
  • brittle；
  • agent 之间上下文传递不完整；
  • 决策权被分散后，难以维持全局一致性；
  • 多个 agent 可能重复劳动或产生冲突；
  • 出现问题后难以定位责任；
  • 长任务中的关键假设和约束容易丢失。

Cognition 的经验总结强调：目前 agents 还不能稳定地进行类似人类工程团队那样的 long-context proactive discourse。

例如两个 coding agents 产生 merge conflict 时，理想情况是：

  1. 双方解释各自的设计；
  2. 识别冲突来源；
  3. 重新协商接口；
  4. 共同更新方案；
  5. 保持对全局目标的一致理解。

但当前系统往往不能比一个强 single agent 更可靠地完成上述长程交流。

已经较好解决的问题

LLM 让不同 agents 可以直接使用自然语言沟通。

过去 multi-agent research 中，一个长期难题是：

如何让 agents 自发形成能够互相理解的 communication protocol？

LLM 使“交流媒介”本身不再是主要障碍。

尚未解决的问题

自然语言可用，不等于协作已经解决。当前更困难的是：

  • 如何选择真正有价值的信息进行传递；
  • 如何维护 shared context；
  • 如何发现并解决分歧；
  • 如何避免错误在 agent 之间传播；
  • 如何让局部任务结果与全局目标一致；
  • 如何使多 agent 系统稳定优于单 agent。

**本节结论：**当前瓶颈已经从“agents 能否交流”转向“agents 能否可靠、主动、长期地协调”。

20.5 值得继续研究的问题

  • 如何构造稳定的 cross-agent shared memory？
  • 如何自动判断任务是否值得并行拆分？
  • 如何进行动态 routing 与 re-routing？
  • 如何让 agents 发现隐含冲突并主动协商？
