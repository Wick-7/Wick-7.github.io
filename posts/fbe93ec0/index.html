训练神经网络的各种技巧

1. 机器学习的三个步骤

训练模型可以分为三个步骤：

  1. 我要什么：定义目标与 Loss Function。
  2. 我有哪些选择：决定函数集合，也就是模型架构。
  3. 选择一个最好的：通过 Optimization 找到参数。

训练完成后，需要先在 Validation Set 上选择模型和超参数，最后才在 Test Set 上进行一次最终评估。

深度学习的本质，是使用神经网络扩大可选择的函数范围。但函数范围更大，并不代表一定能训练成功，也可能更容易过拟合。

2. 先判断问题出在哪里

2.1 Optimization 问题

现象：Training Loss 本身就降不下去。

例如，深层神经网络的表达能力比线性模型更强，但它的 Training Loss 反而高于线性模型。这通常不是 Overfitting，而是没有成功找到神经网络中较好的函数。

此时应该考虑：

  • 更好的 Optimizer；
  • 调整 Learning Rate；
  • Skip Connection；
  • Normalization；
  • 更好的 Initialization 或 Pre-training。

2.2 Generalization 问题

现象：Training Loss 已经很低，但 Validation Loss 较高。

这才是典型的 Overfitting。模型在训练集上表现很好，但无法推广到未见数据。

此时应该考虑：

  • Dropout；
  • 更多训练数据；
  • Data Augmentation；
  • Semi-supervised Learning；
  • Parameter Regularization；
  • Weight Decay 或 AdamW。

不能只因为模型表现不好，就笼统地说是 Overfitting。首先要检查 Training Loss 是否已经足够低。

Part I：改进步骤三——Optimization

3. Vanilla Gradient Descent

目标是寻找：

$$ \theta^*=\arg\min_{\theta}L(\theta) $$

每一步先计算梯度：

$$ g^t=\nabla_\theta L(\theta^t) $$

再更新参数：

$$ \theta^{t+1}=\theta^t-\eta g^t $$

其中，$\eta$ 是 Learning Rate。

Vanilla Gradient Descent 只根据当前梯度决定更新方向，而 Optimizer 通常会综合过去的梯度信息。

4. Learning Rate 为什么难调

Learning Rate 太大时，参数可能越过谷底，甚至发散；太小时，训练又会非常缓慢。

更麻烦的是，不同参数方向的 Loss Surface 可能很不一样：

  • 梯度通常较大的方向，需要较小的 Learning Rate；
  • 梯度通常较小的方向，需要较大的 Learning Rate。

因此，自适应优化器会为不同参数设置不同的有效 Learning Rate。

5. Adagrad

Adagrad 累积每个参数过去梯度的平方：

$$ \sigma_i^t=\sqrt{\sum_{k=0}^{t}(g_i^k)^2} $$

参数更新为：

$$ \theta_i^{t+1} = \theta_i^t- \frac{\eta}{\sigma_i^t+\epsilon}g_i^t $$

直觉：

  • 某一维梯度长期较大，$\sigma_i^t$ 较大，有效 Learning Rate 变小；
  • 某一维梯度长期较小，有效 Learning Rate 较大。

问题

$\sigma_i^t$ 会不断累积，很早以前的梯度和最近的梯度都有影响，导致 Learning Rate 只会越来越小，难以及时适应地形变化。

6. RMSProp

RMSProp 不再累积所有历史梯度，而是对梯度平方做 Exponential Moving Average：

$$ (\sigma_i^t)^2 = \alpha(\sigma_i^{t-1})^2 + (1-\alpha)(g_i^t)^2, \qquad 0<\alpha<1 $$

更新为：

$$ \theta_i^{t+1} = \theta_i^t- \frac{\eta}{\sigma_i^t+\epsilon}g_i^t $$

与 Adagrad 相比，RMSProp 更重视最近的梯度，因此可以更快地调整每个参数的有效 Learning Rate。

7. Momentum

当梯度很小、方向反复震荡，或者经过 Saddle Point 时，Vanilla Gradient Descent 容易停滞。

Momentum 综合过去的梯度方向：

$$ m_i^t = \beta m_i^{t-1} + (1-\beta)g_i^t $$

然后更新：

$$ \theta_i^{t+1} = \theta_i^t-\eta m_i^t $$

直觉上，参数像具有惯性的物体：

  • 梯度长期方向一致时，移动速度会加快；
  • 梯度来回震荡时，互相抵消；
  • 即使当前梯度较小，也可能依靠过去积累的动量继续前进。

8. Adam

Adam 可以理解为：

Momentum + RMSProp

一阶动量：

$$ m_i^t = \beta_1m_i^{t-1} + (1-\beta_1)g_i^t $$

二阶动量：

$$ v_i^t = \beta_2v_i^{t-1} + (1-\beta_2)(g_i^t)^2 $$

简化后的更新式为：

$$ \theta_i^{t+1} = \theta_i^t - \eta \frac{m_i^t}{\sqrt{v_i^t}+\epsilon} $$

实际 Adam 还会对训练初期的一阶、二阶动量进行 Bias Correction。

Adam 通常能够改善 Optimization，使 Training Loss 更容易下降，但不保证 Validation Loss 一定更低。

9. Learning Rate Scheduling

即使使用 Adam，整体 Learning Rate 仍然需要随训练过程变化。

Warm-up

训练初期逐渐增大学习率：

  • 初始参数和动量估计不稳定；
  • 先用小步探索地形，避免训练突然发散。

Learning Rate Decay

训练后期逐渐减小学习率：

  • 前期需要较大的步伐快速探索；
  • 后期接近较优位置时，需要较小步伐稳定收敛。

10. Dropout

训练时随机丢弃一部分神经元；Validation 和 Testing 时使用完整网络。

Dropout 相当于在训练时增加难度，使模型不能过度依赖某几个神经元，从而改善 Generalization。

需要注意：

  • Dropout 通常会让 Training Loss 更高；
  • 它不是用来解决 Training Loss 降不下去的问题；
  • 适合 Training Loss 很低、Validation Loss 较高的 Overfitting 情况。

11. Initialization 与 Pre-training

不同初始参数可能使优化过程走向不同区域，因此即使使用相同模型与 Optimizer，训练结果也可能不同。

对于 ReLU 网络，常见的 Kaiming Initialization 会根据输入维度调整参数尺度，例如：

$$ W_{ij}\sim \mathcal{N} \left( 0,\frac{2}{\mathrm{fan\_in}} \right) $$

合适的 Initialization 可以避免每层数值过大或过小，使梯度传播更加稳定。

Pre-training

先在一个容易获得大量数据的 Pretext Task 上训练，再将参数作为真正 Downstream Task 的初始化。

Pre-training 的作用包括：

  • 提供更好的起点，改善 Optimization；
  • 利用大规模数据学习通用表示，改善 Generalization。

因此，Pre-training 同时可能帮助 Optimization 和 Generalization。

Part II：改进步骤二——模型架构

12. 选择合适的函数范围

  • 函数集合太小：可能根本不包含理想函数，产生 Underfitting。
  • 函数集合太大：虽然表达能力强，但更容易 Overfitting。
  • 理想情况：利用 Domain Knowledge 设计一个范围较小、但刚好包含好函数的架构。

13. Convolutional Neural Network

CNN 利用图像中的两个先验知识。

13.1 Receptive Field

图像中的基础模式，如边缘、眼睛、鸟嘴，只占据局部区域，因此一个神经元不需要连接整张图像。

常见设置包括：

  • Kernel Size；
  • Stride；
  • Padding；
  • 多个 Channel。

局部连接大幅减少参数数量。

13.2 Parameter Sharing

相同模式可能出现在图像的不同位置，因此不同位置可以使用同一组参数，也就是同一个 Filter。

CNN 通过：

  • Local Receptive Field；
  • Parameter Sharing；

缩小函数搜索范围，并把图像的结构先验加入模型，因此主要改善 Generalization。

14. Skip Connection

深层网络中，较前面层的参数变化需要经过很多层才影响 Loss，可能出现：

  • Gradient Vanishing；
  • Gradient Exploding。

Residual Block 将输入直接加到变换结果上：

$$ a'=F(a)+a $$

简化为线性变换时：

$$ a'=Wa+a $$

Skip Connection 为信息和梯度提供一条“高速公路”，使深层网络的 Loss Surface 更容易优化。

它修改了网络架构，但主要带来 Better Optimization。

15. Normalization

Normalization 把每层的输出限制到较稳定的范围，使不同维度的尺度更接近，从而让 Learning Rate 更容易调整。

常见方法包括：

  • Batch Normalization；
  • Layer Normalization；
  • Batch Renormalization。

Normalization 修改了模型架构，主要改善 Optimization，有时也能改善 Generalization。

Part III：改进步骤一——Loss 与数据

16. 为什么不能直接用 Accuracy 当 Loss

分类模型输出每个类别的分数，预测类别通常为：

$$ \hat{c}=\arg\max_i z_i $$

Accuracy 只关心最终类别是否正确。当参数发生小变化但最大类别没有改变时，Accuracy 完全不变，因此大多数位置的梯度为零。

所以 Accuracy 适合 Evaluation，但不适合作为 Gradient Descent 的训练 Loss。

17. Softmax 与 Cross-entropy

Softmax 将类别分数转换为概率：

$$ p_i= \frac{\exp(z_i)} {\sum_j\exp(z_j)} $$

对于 One-hot Ground Truth $\hat{y}$，Cross-entropy 为：

$$ L_{\mathrm{CE}} = -\sum_i \hat{y}_i\log p_i $$

当正确类别为 $c$ 时：

$$ L_{\mathrm{CE}}=-\log p_c $$

Cross-entropy 是连续且可微的，可以为参数更新提供有效梯度。

训练时使用 Cross-entropy，Validation 和 Testing 时仍然可以使用 Accuracy。

18. 更多训练数据

当模型 Overfitting 时，最直接的方法是收集更多具有代表性的训练数据，使训练分布更接近真实测试分布。

但增加数据主要改善 Generalization，并不会让 Optimization 变得更简单。

19. Data Augmentation

当真实数据难以收集时，可以通过合理变换创造新数据。

例如：

  • 图像分类：翻转、裁剪、模糊、颜色变化；
  • 语音识别：调整音量、速度或说话人音色。

变换必须保持任务的 Label 不变：

  • 判断动物类别时，左右翻转通常合理；
  • 判断动物朝向时，左右翻转会改变 Label；
  • 语音识别可以改变说话人音色；
  • 说话人识别则不能随意改变音色。

因此，选择 Augmentation 依赖对任务的理解。

Mixup

选择两笔样本，以比例 $\lambda$ 混合：

$$ \tilde{x} = \lambda x_i+(1-\lambda)x_j $$

同时混合标签：

$$ \tilde{y} = \lambda y_i+(1-\lambda)y_j $$

Mixup 使模型在样本之间学习更平滑的决策边界，主要改善 Generalization。

20. Semi-supervised Learning

现实中通常有少量带标签数据和大量无标签数据，可以定义：

$$ L = \sum_{(x,y)\in D_L}l(f(x),y) + \lambda \sum_{x\in D_U}l'(f(x)) $$

其中第二项利用无标签数据，并依赖对数据结构的假设。

20.1 Entropy Minimization

假设样本应明确属于某个类别，要求无标签样本的预测分布具有较低 Entropy：

$$ H(p) = -\sum_i p_i\log p_i $$

最小化 Entropy 会使预测更加确定，并倾向让分类边界经过低密度区域。

20.2 Graph-based Consistency

假设“相近样本应该具有相近输出”，建立样本图：

$$ L = L_{\mathrm{sup}} + \lambda \sum_{x,x'\in D_U} g(x,x')\, d(f(x),f(x')) $$

其中：

  • $g(x,x')$ 表示两个样本是否相连；
  • $d(\cdot,\cdot)$ 衡量两个输出分布的距离。

Semi-supervised Learning 修改了 Loss，主要改善 Generalization。

21. Parameter Regularization

除了数据 Loss，还可以加入人类对函数的偏好：

$$ L'(\theta) = L(\theta)+\lambda R(\theta) $$

L2 Regularization 定义为：

$$ R(\theta)=\sum_i\theta_i^2 $$

因此：

$$ \nabla_\theta L' = \nabla_\theta L+2\lambda\theta $$

用 Vanilla Gradient Descent 更新：

$$ \theta^{t+1} = \theta^t-\eta \left( \nabla_\theta L+2\lambda\theta^t \right) $$

整理得到：

$$ \theta^{t+1} = (1-2\eta\lambda)\theta^t - \eta\nabla_\theta L $$

可以理解为：每次更新前先把参数缩小一点，这就是 Weight Decay。

Parameter Regularization 与 Weight Decay 主要用于改善 Generalization，而不是改善 Training Loss。

22. AdamW

在 Vanilla Gradient Descent 中，L2 Regularization 可以写成 Weight Decay；但在 Adam 这类自适应 Optimizer 中，直接把 L2 项混进梯度并不等价于真正的 Weight Decay。

AdamW 将参数衰减与 Adam 的梯度更新分开：

$$ \theta^{t+1} = (1-\eta_t\lambda)\theta^t - \eta_t \frac{\hat{m}^t} {\sqrt{\hat{v}^t}+\epsilon} $$

其中，$\lambda$ 的系数定义不同，有时会写成 $2\lambda$。

相较于 Adam：

  • AdamW 主要通过 Weight Decay 改善 Generalization；
  • 它不一定让 Training Loss 更低；
  • 相较于 Vanilla Gradient Descent，它仍保留 Adam 的 Optimization 优势。

23. 方法总表

方法 修改的具体步骤 主要作用 代价 / 不解决什么
Adagrad、RMSProp、Momentum、Adam 改变如何寻找最优函数：利用历史梯度调整参数更新的方向或步长 Better Optimization：使 Training Loss 更容易下降 不是为了改善 Generalization；需要保存额外的优化器状态，并仍需调整 Learning Rate 等超参数
Learning Rate Scheduling 改变如何寻找最优函数：在训练过程中动态调整 Learning Rate Better Optimization：前期快速探索，后期稳定收敛 不直接解决 Overfitting；不合适的 Warm-up 或 Decay 可能使训练过慢或不稳定
Dropout 改变如何寻找最优函数：训练时随机丢弃部分神经元，在受扰动的网络上优化 Better Generalization：降低模型对个别神经元的依赖 Worse Optimization，Training Loss 往往更高；Training Loss 本来就降不下去时不应优先使用
Initialization / Pre-training 改变寻找最优函数的起点：使用更合适的初始参数，而非完全随机开始 Better Optimization；Pre-training 也可能带来 Better Generalization Pre-training 需要额外数据与计算，且 Pretext Task 不一定能迁移到 Downstream Task；不能完全替代合适的 Optimizer
CNN 改变可选择的函数范围：加入 Local Receptive Field 与 Parameter Sharing Better Generalization：利用图像的局部性和平移共享先验，减少参数 依赖任务先验；若局部性与参数共享的假设不成立，可能限制模型能够表达的函数
Skip Connection 改变可选择的函数范围：在网络中加入残差连接，使信息和梯度可以跨层传递 Better Optimization：缓解深层网络难以训练的问题 主要不是为了解决 Overfitting，不保证 Validation Loss 一定更低
Normalization 改变可选择的函数范围：在网络中加入 BatchNorm、LayerNorm 等归一化操作 Better Optimization；有时也能带来 Better Generalization Generalization 的改善并不保证；会增加额外计算，部分方法还依赖 Batch Statistics
Cross-entropy 代替 Accuracy 改变要优化的目标：用连续、可微的 Cross-entropy 作为训练 Loss Better Optimization：提供有效梯度，使 Gradient Descent 能够训练分类模型 不代表 Accuracy 不重要；Accuracy 适合 Evaluation，但不适合作为训练 Loss；不直接保证 Better Generalization
更多训练数据 改变用来定义训练目标的数据：用更多有标签样本计算 Loss Better Generalization：让训练资料更能代表真实分布 需要收集与标注数据，并增加训练成本；不直接解决 Optimizer 或 Training Loss 降不下去的问题
Data Augmentation / Mixup 改变用来定义训练目标的数据：通过合理变换或样本混合扩充训练集 Better Generalization：增加数据多样性，使模型更加稳健 变换必须保持 Label 合理；错误的 Augmentation 会伤害表现，Training Loss 也可能更高
Semi-supervised Learning 改变要优化的目标：在 Loss 中加入无标签数据的 Entropy、Graph Consistency 等约束 Better Generalization：利用大量无标签数据 依赖“低密度分界”“相近样本应有相近输出”等假设；假设错误可能误导模型
Parameter Regularization 改变要优化的目标：在数据 Loss 中加入对参数或函数复杂度的惩罚 Better Generalization：偏好较简单的函数，减少 Overfitting 会使 Training Loss 更难降低；正则化过强会导致 Underfitting
Weight Decay 改变如何寻找最优函数：在每次参数更新时主动缩小参数 Better Generalization 可能带来 Worse Optimization，使 Training Loss 变高；衰减系数过大会导致 Underfitting
AdamW 改变如何寻找最优函数：将 Adam 的自适应更新与 Weight Decay 分开执行 相比 Vanilla Gradient Descent 有 Better Optimization；相比 Adam 有 Better Generalization 需要额外调整 Weight Decay；不保证比 Adam 获得更低的 Training Loss

1. 什么是 AI AgentAI Agent 没有唯一、严格统一的定义。这里讨论的是人类主要给出 goal，而不是逐步告诉 AI 应该执行什么；Agent 需要自己决定一连串 action，并根据环境回传的 observation 持续调整，直到完成目标。 普通的 AI 使用方式通常是“一条指令对应一个回答”。Agent 面对的任务则往往需要多个步骤，环境还可能发生无法预先知道的变化，因此它不能只执行固定脚本，而要在交互过程中不断判断下一步。 最基本的 Agent loop 可以写成：人类给出 goal；Agent 接收 observation；Agent 产生 action；action 改变环境；环境返回新的 observation；循环继续。这里的重点不是某一次回答是否聪明，而是系统能否长期维持一个面向目标的闭环。AlphaGo 可以被视为典型 Agent：goal 是赢棋，observation 是当前棋盘状态，action 是选择落子位置。这个例子也说明 Agent 的描述方式与 Reinforcement Learning 中的 agent–environment ...
